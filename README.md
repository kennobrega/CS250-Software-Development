# CS 250 Module Eight Journal Reflection

## How do I interpret user needs and implement them into a program? How does creating user stories help with this?

I interpret user needs by understanding what problem they are trying to solve and what outcome they expect from the software. Breaking those needs into user stories helps me focus on the user's perspective and prioritize the features that provide the most value. User stories also make it easier to communicate requirements with the rest of the development team.

## How do I approach developing programs? What Agile processes do I hope to incorporate into my future development work?

I approach development by breaking projects into smaller tasks, planning my work before writing code, and testing each feature as it is completed. Throughout this course I learned how Agile development encourages continuous improvement through sprint planning, daily collaboration, sprint reviews, and retrospectives. These are practices I plan to continue using because they make projects easier to manage and allow changes to be incorporated throughout development.

## What does it mean to be a good team member in software development?

Being a good team member means communicating effectively, meeting deadlines, accepting constructive feedback, and helping others when needed. Software development is a collaborative effort, and successful teams rely on trust, accountability, and continuous communication. I believe contributing to team goals while remaining open to learning from others is an important part of becoming a successful software developer.
